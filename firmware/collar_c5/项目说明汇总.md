# leashLink 项目说明汇总

## 1. 项目目标

leashLink 是一个基于 ESP32 的智能遛狗安全系统，重点是实现“本地安全闭环”：

- 感知：项圈端采集运动、加速度、状态
- 判断：手柄端根据张力、距离、运动状态做安全判断
- 执行：手柄锁止、项圈声光震、云端事件上报

## 2. 四份文档分别在讲什么

### 2.1 通信协议
说明设备之间如何通信：

- 项圈端和手柄端通过 ESP-NOW 实现低延迟通讯
- BLE 广播用于配对和发现
- Wi-Fi / MQTT 用于云端联动
- 重点是协议、消息类型、ACK、重传和断链处理

### 2.2 系统架构
说明系统是怎么分层和协同工作的：

- 手柄端负责主控、安全判断、执行器控制和云端上传
- 项圈端负责 IMU 感知、反馈执行和低功耗广播
- 云平台只做记录、配置和远程查看，不参与高实时闭环

### 2.3 硬件接口
说明需要购买和接线的硬件：

- 手柄端：ESP32-C3、HX711、GPS、OLED、舵机、蜂鸣器、震动马达
- 项圈端：ESP32-C5、IMU、LED、蜂鸣器、震动马达、电池管理

### 2.4 API 文档
说明后端和设备之间的接口定义：

- 设备注册
- 实时状态查询
- 遥测上传
- 事件上传
- 围栏配置

## 3. 你们项目当前最适合的开发路线

### 固件端
- 统一使用 ESP-IDF
- 目标是先做最小可用版本：
  1. IMU 采集
  2. 运动状态识别
  3. ESP-NOW 遥测发送
  4. 手柄控制命令接收
  5. 声光震反馈

### 后端端
- 使用 Python + FastAPI
- 先实现内存版接口，后续再接数据库与 MQTT

## 4. VS Code 没有 ESP-IDF 扩展能不能运行？

### 结论
- 不能“只靠扩展”运行项目，但也不是必须安装扩展才能开发。
- 如果你想直接用 VS Code 里最顺手的方式编译、烧录、串口调试，安装 ESP-IDF 扩展会非常方便。

### 具体分为两种情况

1. 只想看代码、改逻辑
- 不安装扩展也可以
- 但你需要单独安装 ESP-IDF 工具链
- 例如 Espressif IDF 安装包、Python、CMake、Ninja 等

2. 想真正编译、烧录、串口监视
- 最推荐安装 ESP-IDF 扩展
- 否则需要手动在终端里使用 idf.py build / flash / monitor

## 6. 最推荐的准备步骤

### 方案 A：最省事
- 安装 VS Code 的 ESP-IDF 扩展
- 让扩展自动帮你配置工具链

### 方案 B：手动配置
- 安装 Python 3.13
- 安装 ESP-IDF 的官方工具链
- 配置 PATH
- 运行 idf.py build

## 7. 你现在最该做的事

如果你要继续做这个项目，建议按这个顺序推进：

1. 先确认硬件清单是否和文档一致
2. 先把项圈端最小功能做出来：IMU + ESP-NOW + LED/Buzzer/Vibration
3. 再继续配 ESP-IDF 环境
4. 最后再做手柄端联调
